# Week 1 — Git & GitHub

## What Git is

Git is a version control system. It tracks every change to every file in a project.
It lets you: go back in time, work in parallel branches, collaborate without overwriting each other.

## Core commands I'm learning

- git init / git clone — start tracking a project
- git status — see what's changed
- git add — stage changes for commit
- git commit — save a snapshot with a message
- git push — send commits to GitHub
- git pull — get latest changes from GitHub
- git branch — create a parallel version of the project
- git checkout — switch branches
- git merge — combine branches

## What Branches are

It allows to work on the bug fixes or editing the code without having an influence on the main code so you could work in parallel. It's like creating a copy of a project where I can experement and make changes independently.
